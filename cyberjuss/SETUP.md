# GitHub profile README setup (cyberjuss)

This folder (**cyberjuss**) has your **profile README** in the style of [tayontech](https://github.com/tayontech). It will show on your GitHub profile when the repo is set up correctly.

## Steps

1. **Create a new repo on GitHub** named exactly **`cyberjuss`** (same as your GitHub username).  
   - Public, no need to add a .gitignore or license for an empty start.

2. **Copy `README.md` and `Duru_Justin_Resume.pdf`** from this folder into the root of that repo.

3. **Edit the placeholders** in `README.md`:
   - Location
   - Contact email
   - LinkedIn / X (Twitter) URLs
   - Any bio or "learning" lines

4. **Push.** Your profile at `https://github.com/cyberjuss` will show this README.

## Using the lab repo under cyberjuss

If your lab repo will live under the **cyberjuss** account:

- Create (or rename) the repo to **AWS-Threat-Detection-Soc-Lab** (display title can be **AWS Threat Detection Soc Lab**) under **cyberjuss**.
- In your local lab folder, set the remote (or add a second remote):
  ```bash
  git remote set-url origin https://github.com/cyberjuss/AWS-Threat-Detection-Soc-Lab.git
  ```
- The profile README already links to `https://github.com/cyberjuss/AWS-Threat-Detection-Soc-Lab`.
